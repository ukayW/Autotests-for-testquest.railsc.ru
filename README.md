## 📋 О проекте

В рамках данного тестового задания реализован набор из **5 end-to-end автотестов** для проверки функционала авторизации пользователя. Проект демонстрирует навыки работы с современными инструментами автоматизации тестирования.

### ✅ Покрытые сценарии

1.  **Успешная авторизация:** Проверка успешного входа в личный кабинет с валидными учетными данными.
2.  **Ввод email с пробелами в начале и конце:** Попытка входа с верным email и паролем, но с пробелами в email. Проверка тримминга данных.
3.  **Неверный пароль:** Проверка входа с существующим email и неверным паролем.
4.  **Вход с несуществующим email:** Ввод несуществующего адреса электронной почты. Проверка блокировки отправки формы или сообщения об ошибке.
5.  **Пустой ввод данных:** Отправка формы без заполнения полей email и пароля. Проверка валидации полей.


## ⚙️ Технологический стек

*   **Playwright** — фреймворк для кросс-браузерного E2E-тестирования.
*   **TypeScript** — типизированный язык для повышения надежности кода.
